## Документация проекта Fleur de Rêve

### Эта папка содержит вспомогательную документацию интернет‑магазина цветов **Fleur de Rêve**.

---

### Features

- 🏵️ **Каталог цветов и букетов**  
  Готовые букеты, цветы поштучно, цветы в коробке, комнатные растения, подарочные наборы, сезонные предложения и акции.

- 🔍 **Фильтрация и поиск**  
  Фильтрация по категории, поводу, цвету, цене, поиск по названию.

- 🛒 **Корзина и оформление заказа**  
  Добавление товаров в корзину, изменение количества, удаление позиций, расчёт итоговой стоимости, оформление заказа с выбором способа оплаты и доставки.

- 👤 **Регистрация и авторизация**  
  Регистрация новых пользователей, вход/выход из аккаунта, редирект на главную после успешного входа.

- 📦 **История заказов и профиль**  
  Просмотр последних заказов в личном кабинете и отдельная страница истории всех заказов.

- 🌸 **Заявка на авторский букет**  
  Форма с указанием повода, предпочитаемых цветов, цветовой гаммы, бюджета и пожеланий.  
  Заявки сохраняются в базе данных с разными статусами обработки.

- ⭐ **Отзывы клиентов**  
  Добавление отзывов с оценкой (1–5 звёзд), модерация отзывов через админ‑панель, расчёт средней оценки и процента рекомендаций.

- 📚 **Информационные страницы**  
  Условия доставки, гайды по уходу за цветами, контакты и информация о магазине.

- ⚠️ **Обработчики ошибок**  
  Кастомные страницы 400, 403, 404, 500.

- 📊 **Административная панель**  
  Управление товарами, заказами, отзывами и заявками на авторские букеты через Django Admin.

---

### Tech Stack

- **Backend**: Python 3.10+, Django 5.0, SQLite  
- **Frontend**: HTML5, CSS3 (включая кастомную тему `moon-style.css`), JavaScript (Vanilla), Django Templates  
- **Tools**: Django Admin, SQLite

### Installation

#### Предварительные требования

- **Python** 3.10 или выше  
- **pip** (менеджер пакетов Python)  
- (Опционально) **Git** для клонирования репозитория

#### Пошаговая установка

1. **Клонируйте репозиторий и перейдите в папку проекта:**
```
   git clone <repository-url>
   cd flowershop
```

2. **Создайте и активируйте виртуальное окружение:**

   ```
    Windows   python -m venv venv   venv\Scripts\activate
    Linux/Mac   python3 -m venv venv   source venv/bin/activate
   ```
   
3. **Установите зависимости:**
   
```
   pip install -r requirements.txt
```
   
5. **Примените миграции:**
   
```
   python manage.py migrate

```

7. **Создайте суперпользователя (опционально, для доступа к админ‑панели):**
   
```
   python manage.py createsuperuser
```
   
9. **Запустите сервер разработки:**
```
   python manage.py runserver
```  
   
11. **Откройте браузер и перейдите по адресу:**
   
http://127.0.0.1:8000/

### Описание моделей для твоего проекта
---

- **User** — пользователи интернет‑магазина (покупатели), которые могут регистрироваться, авторизоваться, оформлять заказы и оставлять отзывы.
- **Category** — категории товаров (типы букетов и товаров: готовые букеты, цветы поштучно, коробки, растения, подарки, акции и т.д.), используются для группировки и фильтрации каталога.
- **Product** — товар каталога (букет, композиция, растение, подарок): название, цена и старая цена (для акций), тип категории, повод, цвет, описание, изображение, флаг наличия и дата создания.
- **Cart** — позиции корзины пользователя: какой пользователь, какой товар, в каком количестве и когда добавлен; используется для страницы корзины и расчёта итоговой суммы.
- **Order** — оформленный заказ: связь с пользователем, контактные данные (имя, телефон, email, адрес), дата и время доставки, способ оплаты, пожелания, итоговая стоимость и статус заказа (новый, в обработке, передан курьеру, выполнен, отменён).
- **CustomBouquetRequest** — заявка на авторский (индивидуальный) букет: повод, предпочитаемые цветы, цветовая гамма, бюджет, пожелания клиента, его контактные данные, статус обработки заявки и отметки времени создания/обновления.
- **Review** — отзыв клиента о магазине: опциональная связь с пользователем, имя автора, текст отзыва, оценка в звёздах, флаг «одобрен модератором» и дата создания; используется для страницы отзывов и расчёта средней оценки.

### Компоненты системы 
---
1. **Браузер (Client)**
  Отображает страницы магазина (HTML/CSS), выполняет небольшой JavaScript, отправляет HTTP‑запросы и формы (регистрация, логин, корзина, заказ, отзывы).

2. **Django Backend**
  Принимает запросы, маршрутизирует их через urls.py, выполняет бизнес‑логику во views.py, обращается к БД через модели (models.py) и рендерит HTML‑шаблоны.

3. **Database (SQLite)**
  Хранит пользователей, категории, товары, содержимое корзины, оформленные заказы, отзывы и заявки на авторские букеты.
4. **Django Admin**
  Административная панель для управления товарами, заказами, отзывами, заявками и пользователями.

### Поток данных 
---

1. Пользователь открывает страницу — браузер отправляет HTTP‑запрос к Django.
2. Django‑view обрабатывает запрос, при необходимости читает/пишет данные в SQLite через ORM.
3. Django рендерит HTML‑шаблон с данными и возвращает его в браузер.
4. При действиях (корзина, заказ, отзывы, регистрация) браузер отправляет POST‑формы, бэкенд обновляет БД и делает редирект на страницу с обновлённым состоянием.
