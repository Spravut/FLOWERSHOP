# Автопроверка отзывов

Перед сохранением отзыва вызывается `app/review_validation.py`:

- **Ссылки и спам:** `http(s)://`, `www.`, типичные мессенджеры/соцсети, короткие ссылки, e-mail, IPv4, домены с популярными TLD.
- **Ненормативная лексика:** список корней/слов в `_PROFANITY` (дополняйте под проект).
- **Повтор символов:** 10+ одинаковых символов подряд.

При успехе отзыв создаётся с `is_approved=True` и сразу попадает в выдачу `GET /reviews?approved_only=true`.

При нарушении — **422** и текст в `detail`.

Старые записи в БД с `is_approved=false` не меняются автоматически. При необходимости одобрить вручную в БД:

```sql
UPDATE reviews SET is_approved = true WHERE is_approved = false;
```

(используйте только если доверяете данным).
